ThreadLocal - это класс в Java, предоставляющий механизм для создания переменных, которые будут иметь разные
значения для каждого потока. Это очень полезный инструмент при работе с многопоточностью, когда вам нужно иметь
данные, специфичные для каждого потока, но не хотите передавать их явно между методами или классами.
Основные методы и функции ThreadLocal:
    get() и set(T value): Метод get() возвращает значение переменной, связанной с текущим потоком. Метод set(T value)
    устанавливает значение переменной для текущего потока.
    initialValue(): Этот метод предоставляет способ инициализации значения переменной для каждого потока.
    Вы можете переопределить этот метод, чтобы предоставить начальное значение.
    remove(): Этот метод удаляет значение переменной, связанное с текущим потоком. Это может быть полезным,
    чтобы избежать утечек памяти.
В этом примере создается ThreadLocal переменная threadLocal, связанная с каждым потоком. При каждом выполнении потока
значение этой переменной увеличивается на 1. Поскольку каждый поток имеет свою собственную копию переменной, изменения
в одном потоке не затрагивают другие потоки.
ThreadLocal особенно полезен, когда вы хотите сохранить данные, такие как соединения с базой данных, сессии пользователя
или контекст выполнения, которые будут иметь разные значения для каждого потока. Однако стоит быть осторожными с
использованием ThreadLocal, так как он может привести к утечке памяти, если не правильно управлять жизненным циклом
переменных для каждого потока.
